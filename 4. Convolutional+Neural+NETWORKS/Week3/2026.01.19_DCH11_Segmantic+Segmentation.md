# Region Proposals: R-CNN and its Evolution

## 1. The Motivation: Smarter Scanning
Sliding windows (even with convolutional implementation) scan *every* part of the image, including large areas of empty sky or road.
**Region Proposals** aim to be smarter: instead of scanning everywhere, select only a few thousand "candidate" regions that *look* like they might contain an object (blobs, textures) and run the classifier only on those.


## 2. The R-CNN Algorithm (Regions with CNN features)
Proposed by Girshick et al. (2013), this method follows these steps:
1.  **Propose Regions:** Run a segmentation algorithm (like Selective Search) to find ~2000 "blobs" or regions of interest.
2.  **Classify:** Run a ConvNet on each proposed region independently.
3.  **Refine:** Output a label (e.g., Car) and a corrected bounding box ($b_x, b_y...$) for that region.
* *Drawback:* It is **slow** because it classifies 2000 regions sequentially.

## 3. Faster Algorithms: The Evolution
To fix the speed issues, the algorithm evolved:

* **Fast R-CNN (2015):** Instead of running the ConvNet 2000 times, it uses a **Convolutional Implementation** (similar to what we learned earlier). It runs the ConvNet *once* on the image to get a feature map, then uses "RoI Pooling" to extract features for the specific regions from that map.
* **Faster R-CNN (2016):** The bottleneck was now the region proposal step itself (segmentation is slow). Faster R-CNN replaces the segmentation algorithm with a **Region Proposal Network (RPN)**â€”a neural network trained to propose regions. This makes the whole pipeline much faster.


## 4. R-CNN vs. YOLO
* **R-CNN Family:** Two-step process (Propose Region $\to$ Classify). Historically more accurate but slower.
* **YOLO:** One-step process (Look Once $\to$ Detect everywhere). Generally much faster and better for real-time applications.
* *Note:* While R-CNN is influential, YOLO-style single-shot detectors are often preferred for speed.
